MirageKit Research Program

MirageKit

Your context compression looks valid. It isn't.

When LLM agents compress long conversations, they can silently lose track of which task they're solving while still producing confident answers. We call this a validity mirage. MirageKit is the research program; dreams is the public showcase for papers and artifacts; the evaluation MCP lives in the separate tropical-mcp repository.

Read this like a research packet: flagship paper, replay witness, certificate artifact, and a reproducible three-call verification path.

Start with the flagship paper, inspect the replayed witness, then run the local verify path to reproduce the divergence yourself.

Codex Quick-Start

Clone the implementation repo, then register the MCP in Codex.

git clone https://github.com/jack-chaudier/tropical-mcp.git ~/tropical-mcp
codex mcp add tropical-mcp --env TROPICAL_MCP_CLIENT=codex -- \
  uv --directory ~/tropical-mcp run tropical-mcp
codex mcp list

Minimal Verification

Use an explicit test payload so the pivot and predecessor structure is observable.

messages = [
  {"id":"goal","role":"user","content":"Build a long-running coding agent workflow for Codex.","role_hint":"pivot"},
  {"id":"constraint_stdio","role":"user","content":"Use stdio transport and never emit JSON-RPC data to stdout logs.","role_hint":"predecessor"},
  {"id":"constraint_clients","role":"user","content":"Support Codex and Claude-style clients through explicit MCP tool calls.","role_hint":"predecessor"},
  {"id":"status","role":"assistant","content":"I am wiring the verification flow and docs.","role_hint":"noise"}
]

runtime_info()
compact_auto(messages=messages, token_budget=45, k_target=2, mode="adaptive")
certificate(messages=messages, token_budget=45, k=2)
Expected signals

runtime_info() should report the resolved client and telemetry path. compact_auto(...) should choose l2_guarded on the sample. certificate(...) should show comparable recency vs guarded kept/dropped IDs.

VALIDITY MIRAGE DETECTED

Raw validity still passes, but the causal anchor is gone. Example: the model can still answer an NTSB query, but now about the wrong incident after the original root-cause pivot was evicted.

Math Snapshot

Core Contract

d_pre >= k

Guarded compaction is certified safe only when the pivot retains its required predecessor depth.

Frontier Feasibility

W[k] = infinity -> infeasible

If the k-slot frontier is infinite, no valid completion exists for that retained context.

Raw Validity

raw = max(primary_full, decoy_full)

Answerability alone can stay high even when pivot identity has silently changed.

Mirage Gap

delta = raw - pivot_preservation

Large positive gap indicates a validity mirage regime rather than true semantic stability.

Exp58 Deterministic Witness

The frontier vector W[0..k] measures whether the pivot and its k nearest predecessors can all survive a compaction pass. When W[k] = ∞, no valid configuration exists under that policy — the predecessors were already evicted. A finite W[k] means the guard found a feasible retention set.
